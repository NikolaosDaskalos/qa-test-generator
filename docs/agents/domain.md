# Domain Docs

This is a single-context repository.

## Required Reading

Before working in the domain, read:

- `CONTEXT.md` for canonical domain language
- Relevant ADRs under `docs/adr/`, when present
- `docs/backend-plan.md` for the current backend implementation plan

Proceed silently when no relevant ADR exists.

## Vocabulary

Use terms defined in `CONTEXT.md` in plans, issues, implementation, and tests. Avoid synonyms explicitly listed under `_Avoid_`.

If a necessary domain concept is missing, note it for a future `grill-with-docs` session rather than inventing conflicting language.

## ADR Conflicts

Surface any conflict with an existing ADR explicitly instead of silently overriding it.
